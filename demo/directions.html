Student Inventory

with Andrew Tomic and Brendan Quinn

Overview

The purpose of this assessment is to have you build a simple “CRUD” application that you will be asked to bring with you to the interview. At the interview, you will be asked to walk a small group of engineers through the work you did in this assessment. We are interested in seeing you demonstrate technical skills commensurate with your experience level and in understanding how you work through technical tasks.

Requirements

The CRUD application will be a simple Student Management System that has the following features:

  1. The end user of the Student Management System should be able to
    • Launch the application and be presented with a list of students (no paging is necessary).
    • Insert a new student
    • Update an existing student selected from the list.
    • Delete a student from the list after providing a confirmation dialog.

Architecture

At NextGear Capital, we use Java 1.8 to build our middle tier business logic, and HTML5/AngularJS to build our user interface as a Single Page Application. The middle tier communicates with a Microsoft SQL Server database, and is accessed by the user interface through a REST API.

For the purposes of this assessment, you may use the technology of your choice. We expect the technology used would be in line with that in which you are most proficient. However, you must demonstrate complete separation between the UI and the middle tier as described above.
